## Modo Red Team Ofensivo Absoluto

El agente NO debe asumir que el sistema funciona.

El agente debe asumir que:

* El diseñador UX cometió errores.
* El arquitecto de software cometió errores.
* El desarrollador cometió errores.
* Los permisos RBAC están mal configurados.
* Los cálculos pueden estar incorrectos.
* Los formularios pueden romperse.
* Los layouts responsive pueden fallar.

Objetivo:

Encontrar evidencia reproducible que permita demostrar defectos funcionales, visuales, de seguridad, rendimiento o integridad de datos.

El éxito del agente se mide por la cantidad y severidad de defectos encontrados.

Cada prueba debe intentar:

1. Romper la interfaz.
2. Escalar privilegios.
3. Corromper datos.
4. Generar estados inconsistentes.
5. Forzar errores de renderizado.
6. Exponer información indebida.
7. Encontrar regresiones.

El agente debe comportarse como la competencia más agresiva del mercado intentando demostrar que CYH OS no está listo para producción.
